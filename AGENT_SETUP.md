# AgentShield — Agent Setup Guide

> **This document is designed to be read and executed by an AI agent.**
> Follow the steps sequentially. Each step has a clear action, verification command, and expected output.
> If a step fails, go to the [Troubleshooting](#troubleshooting) section before continuing.

---

## Overview

You are setting up AgentShield, a security layer that will intercept package installation requests and check them for vulnerabilities before they execute. After setup, every `pip install`, `npm install`, or `cargo add` your agent makes will be scanned automatically.

**What you will do:**
1. Install AgentShield
2. Create a configuration file with sensible defaults
3. Set up API keys (optional but recommended)
4. Warm the local vulnerability database
5. Wire AgentShield into the active agent framework
6. Verify the installation works

**Time estimate:** 5–10 minutes (plus ~5 minutes for cache warm).

**Prerequisites:**
- Python 3.11 or later installed (`python3 --version` must show 3.11+)
- pip available (`pip --version`)
- Internet access for the initial cache warm

## Step 1 — Install AgentShield

Run the installation command appropriate to your environment.

**Standard install (CVE checks + typosquatting only):**
```bash
pip install git+https://github.com/mkarvan/AgentShield.git
```

**With static analysis support (`--deep` flag):**
```bash
pip install "agentshield[static-analysis] @ git+https://github.com/mkarvan/AgentShield.git"
```

**With Hermes Agent integration:**
```bash
pip install "agentshield[hermes] @ git+https://github.com/mkarvan/AgentShield.git"
```

**With OpenClaw integration:**
```bash
pip install "agentshield[openclaw] @ git+https://github.com/mkarvan/AgentShield.git"
```

**With everything:**
```bash
pip install "agentshield[all] @ git+https://github.com/mkarvan/AgentShield.git"
```

**Verify installation:**
```bash
agentshield --version
```
Expected output: `agentshield, version 0.7.0` (or higher).

If `agentshield` is not found, the pip bin directory may not be on PATH. Try:
```bash
python3 -m agentshield --version
```

## Step 2 — Create configuration file

Create the directory and configuration file:

```bash
mkdir -p ~/.config/agentshield
```

Write the following to `~/.config/agentshield/config.toml`:

```toml
# AgentShield configuration — generated by AGENT_SETUP.md

[defaults]
critical = "block"
high     = "warn_confirm"
medium   = "async_report"
low      = "ignore"
info     = "ignore"

[ecosystems.pypi]
high = "block"

[ecosystems.npm]
critical = "block"
high     = "warn_confirm"

[rules]
  [rules."T1.1"]
  mode = "block"

  [rules."T1.2"]
  mode = "block"

  [rules."T3.5"]
  mode = "block"

  [rules."T4.1"]
  mode = "warn_confirm"

[allowlist]
packages = []

[denylist]
packages = []

[api]
nvd_api_key  = ""
github_token = ""

[cache]
db_path   = "~/.agentshield/agentshield.db"
ttl_hours = 24

[reporting]
report_dir          = "~/.agentshield/reports/"
auto_report_on_exit = true

[rate_limits]
max_packages_per_hour    = 20    # block if agent installs > 20 packages/hour
max_wheel_mb_per_session = 500   # block if total wheel downloads exceed 500 MB
```

**Verify the config loads without errors:**
```bash
agentshield cache stats
```
Expected output: lines showing cache entry counts (all zeros on first run). No error messages.

## Step 3 — Set up API keys (recommended)

API keys are optional but improve coverage. Without them AgentShield still works — OSV has no rate limit and covers most packages. Keys are needed for:
- **NVD:** higher rate limit (5 req/30s → 50 req/30s) and CPE-level matching
- **GitHub Advisory:** access to the GitHub Advisory Database (required for some npm/Go vulnerabilities)

### Option A — Environment variables (recommended for agents)

Add to your shell profile (`~/.zshrc`, `~/.bashrc`, etc.) or set in the current session:

```bash
export NVD_API_KEY="your-nvd-api-key-here"
export GITHUB_TOKEN="ghp_your-github-token-here"
```

### Option B — config.toml

Edit `~/.config/agentshield/config.toml` and fill in the `[api]` section:

```toml
[api]
nvd_api_key  = "your-nvd-api-key-here"
github_token = "ghp_your-github-token-here"
```

### How to obtain keys

**NVD API key:**
- Register at: https://nvd.nist.gov/developers/request-an-api-key
- Keys are free. Email confirmation takes a few minutes.
- If the user does not have one yet, skip this — OSV covers most cases.

**GitHub token:**
- Any classic Personal Access Token works — no scopes needed.
- Create at: https://github.com/settings/tokens → "Generate new token (classic)"
- No checkboxes need to be selected; just set a name and expiry.

**Verify keys are detected:**
```bash
agentshield scan requests --ecosystem pypi
```
If keys are set, the scan will use NVD and GitHub Advisory in addition to OSV. The output will show findings from multiple sources if any exist.

## Step 4 — Warm the local cache

This step downloads OSV vulnerability bulk exports and populates the local SQLite database. It enables fast offline scans and makes the posture report useful immediately.

```bash
agentshield cache warm
```

This takes 2–5 minutes on a typical connection. You will see a progress bar.

**Expected output:**
```
Warming cache for: pypi, npm, cargo
  DB path: ~/.agentshield/agentshield.db
  This downloads OSV bulk exports and may take up to 5 minutes.
Cache warm-up complete in 142.3s
  Ecosystems : pypi, npm, cargo
  Advisories : 184231 scanned
  CVE mirror : 12847 entries inserted
  Malicious  : 389 packages recorded
```

**Verify the cache is populated:**
```bash
agentshield cache stats
```
Expected: CVE mirror and malicious DB counts should be non-zero.

**To warm only specific ecosystems (faster):**
```bash
agentshield cache warm --ecosystems pypi          # Python only
agentshield cache warm --ecosystems pypi,npm      # Python + JavaScript
```

### Framework: Hermes Agent

**Prerequisites:** `pip install "agentshield[hermes] @ git+https://github.com/mkarvan/AgentShield.git"` (done in Step 1 if you chose `[hermes]` or `[all]`).

Locate your Hermes configuration file (typically `hermes_config.yaml` or `~/.hermes/config.yaml`).

Add the AgentShield plugin to the `plugins` list:

```yaml
plugins:
  - module: agentshield.integrations.hermes
    class: AgentShieldPlugin
    config:
      config_path: ~/.config/agentshield/config.toml
```

**Restart the Hermes agent** for the plugin to take effect.

#### How interception works

AgentShieldPlugin intercepts two categories of tool calls:

**1. Structured install tools** — `pip_install`, `npm_install`, `cargo_add`

The plugin reads the package name/version directly from the tool call arguments and scans it before the install proceeds.

**2. Shell/bash tools** — `bash`, `shell`, `run_command`, `execute`, `terminal`

Hermes sometimes uses raw bash commands instead of structured install tools (e.g. `pip install --break-system-packages requests`). The plugin automatically parses the command string for any of the following patterns and scans each detected package before allowing the command to run:

| Command pattern | Ecosystem |
|-----------------|-----------|
| `pip install …`, `pip3 install …` | PyPI |
| `python -m pip install …`, `uv pip install …` | PyPI |
| `npm install …`, `npm i …` | npm |
| `yarn add …` | npm |
| `cargo add …`, `cargo install …` | Cargo |

Flags like `--break-system-packages`, `--user`, and `-U` are correctly handled. Packages declared in a referenced requirements/constraint file (`-r requirements.txt`, `-c constraints.txt`) are resolved and scanned too; a remote reference such as `-r https://…/req.txt` is blocked because its contents cannot be verified statically. Version specifiers and extras (`requests[security]==2.28.0`) are stripped to extract the bare package name.

If Hermes uses a different tool name for shell execution, add it to the `intercepts` list by subclassing:

```python
from agentshield.integrations.hermes.plugin import AgentShieldPlugin, _SHELL_TOOLS

class MyAgentShieldPlugin(AgentShieldPlugin):
    intercepts = [*AgentShieldPlugin.intercepts, "run_bash", "exec_cmd"]
```

**Verify:**
Ask the Hermes agent to install a package. It should pause and either allow, warn+confirm, or block depending on your config.

To test with a known-safe package:
```
<hermes agent> Please install the 'requests' package.
```
Expected: AgentShield scans, finds no issues (or it is on your allowlist), allows the install.

To test blocking:
```
<hermes agent> Please install the 'colouredlogs' package.
```
Expected: AgentShield blocks with a message about the package being on the denylist (if you added it) or flags a finding.

To test that bash command interception works:
```
<hermes agent> Run: pip install --break-system-packages colouredlogs
```
Expected: AgentShield parses the bash command, detects `colouredlogs`, and blocks it.

### Framework: Any MCP-compatible agent (Claude, others)

This is the recommended path if your framework is not Hermes or OpenClaw. Any agent that supports MCP (Model Context Protocol) can use AgentShield as an MCP tool server.

**Step 5a — Start the MCP server**

The MCP server runs as a subprocess; your MCP client manages its lifecycle. You do not need to start it manually.

**Step 5b — Add to your MCP client configuration**

For Claude Desktop / Claude Code, add to your MCP settings file:

```json
{
  "mcpServers": {
    "agentshield": {
      "command": "agentshield",
      "args": ["serve", "--mcp"]
    }
  }
}
```

**Claude Code config location:** `~/.claude/claude_desktop_config.json`  
**Claude Desktop config location:** `~/Library/Application Support/Claude/claude_desktop_config.json` (macOS)

For other MCP clients, consult the client's documentation for adding stdio MCP servers; use `agentshield serve --mcp` as the command.

**Step 5c — Restart your MCP client**

The MCP server starts automatically when the client connects.

**Verify:**
Ask the agent to call the `agentshield_scan` tool:
```
Call the agentshield_scan tool with package="requests", ecosystem="pypi"
```
Expected response: JSON with `"decision": "ALLOW"` and an empty or populated findings list.

**Available MCP tools:**

| Tool | When to call |
|------|-------------|
| `agentshield_scan` | Before any pip install / npm install / cargo add |
| `agentshield_scan_file` | To scan all packages in a requirements.txt / package.json / Cargo.toml at once |
| `agentshield_posture` | To get a full security posture report |
| `agentshield_sbom` | To generate a CycloneDX v1.4 JSON SBOM from a manifest file |
| `agentshield_diff_scan` | To scan only changed packages between two manifest snapshots (CI delta scanning) |
| `agentshield_scan_docker` | To scan packages from `RUN pip/npm/cargo install` in a Dockerfile |

**Important — instruct your agent to use AgentShield:**

After connecting AgentShield as an MCP server, add an instruction to your agent's system prompt:

```
Before installing any package with pip, npm, or cargo, you MUST call the
agentshield_scan tool first. If the result is BLOCK, do not install the package
and explain why. If the result is NEEDS_CONFIRMATION, show the findings to the
user and wait for approval. If the result is ALLOW or LOG_ASYNC, proceed.

When working with a manifest file (requirements.txt, package.json, Cargo.toml),
use agentshield_scan_file instead — it scans all packages in the file at once.
If any package is BLOCKED, do not proceed with the bulk install.

To generate a Software Bill of Materials for a project, call agentshield_sbom
with the path to the manifest file. It returns CycloneDX v1.4 JSON.

Pass context_hint to agentshield_scan to explain why you want the package.
```

### Framework: Shell wrapper (any agent)

This approach works with **any agent that runs shell commands** — no framework-specific integration needed. Instead of calling `pip install` directly, the agent (or user) calls a wrapper script that scans with AgentShield first.

This pattern was validated in production with Hermes Agent, which created and used it autonomously during live testing.

**Step 5a — Create the wrapper script**

Save the following to `~/safe-pip-install.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

PACKAGE="$1"
shift  # remaining args passed to pip install

echo "=== AgentShield: scanning $PACKAGE ==="
if ! agentshield scan "$PACKAGE" --ecosystem pypi; then
    echo "=== BLOCKED: AgentShield found security issues with $PACKAGE ==="
    exit 1
fi

echo "=== Scan passed. Proceeding with pip install $PACKAGE $* ==="
exec pip install "$PACKAGE" "$@"
```

```bash
chmod +x ~/safe-pip-install.sh
```

**Step 5b — Add shell aliases (optional)**

Add to `~/.bashrc` or `~/.zshrc`:

```bash
# AgentShield guards — always scan before install
alias safe-pip-install='~/safe-pip-install.sh'
```

**Step 5c — Equivalent wrappers for npm and cargo**

For npm:
```bash
#!/usr/bin/env bash
set -euo pipefail
PACKAGE="$1"; shift
agentshield scan "$PACKAGE" --ecosystem npm || { echo "BLOCKED by AgentShield"; exit 1; }
exec npm install "$PACKAGE" "$@"
```

For cargo:
```bash
#!/usr/bin/env bash
set -euo pipefail
PACKAGE="$1"; shift
agentshield scan "$PACKAGE" --ecosystem cargo || { echo "BLOCKED by AgentShield"; exit 1; }
exec cargo add "$PACKAGE" "$@"
```

**Step 5d — Instruct your agent to use the wrapper**

Add to your agent's system prompt or instructions:

```
When installing packages, always use the safe-install wrappers:
- Python: ~/safe-pip-install.sh <package> [pip flags...]
- npm: ~/safe-npm-install.sh <package> [npm flags...]
- cargo: ~/safe-cargo-add.sh <package> [cargo flags...]

If the wrapper exits with code 1, do NOT proceed with the install.
Report the security findings to the user.
```

**Why this works well:**
- No plugin/MCP setup required — pure shell
- Works with any agent that can run bash commands
- The agent doesn't need to know about AgentShield — the wrapper handles everything
- Easy to audit and customize

## Step 6 — Verify the full setup

Run these verification commands to confirm everything is working.

**6a. Basic scan (online):**
```bash
agentshield scan requests==2.28.0 --ecosystem pypi
```
Expected: Output showing ALLOW or findings. Should complete in < 5 seconds.

**6b. Offline scan (tests cache warm worked):**
```bash
agentshield scan requests==2.28.0 --ecosystem pypi --offline
```
Expected: ALLOW with `cache_hit: False` or findings from CVE mirror. Should complete in < 200 ms.

**6c. Posture report:**
```bash
agentshield posture --skip-packages
```
Expected: A formatted posture report. If async log is empty, the report will say "No async-report log entries in the last 24 hours." — that is correct for a fresh install.

**6d. HTML report:**
```bash
agentshield posture --format html --output /tmp/posture_test.html && echo "HTML written OK"
```
Expected: "HTML written OK" and a file at `/tmp/posture_test.html`.

**6e. IPC daemon (optional, for shell integration):**
```bash
# In one terminal:
agentshield serve &
SHIELD_PID=$!

# In another terminal (or after a brief pause):
sleep 1
echo '{"jsonrpc":"2.0","method":"ping","params":{},"id":1}' | nc -U ~/.agentshield/agentshield.sock

kill $SHIELD_PID
```
Expected: `{"jsonrpc":"2.0","id":1,"result":{"pong":true}}`

**Note on authentication:** On Linux and macOS the server uses peer credential checks (same UID) automatically — `nc` will work without any special handshake. On other platforms a shared-secret token is written to `~/.agentshield/ipc.token` (mode 0o600); clients must send `AUTH <token>\n` before the first JSON-RPC message.

**6f. Manifest file scan:**
```bash
echo -e "requests\nflask" > /tmp/test-req.txt
agentshield scan-file /tmp/test-req.txt
```
Expected: A Rich summary table listing `requests` and `flask`, each with their scan status (ALLOW or findings). Exit code 0 if neither is blocked.

## Quick-reference cheat sheet

Once set up, here are the commands you will use regularly:

```bash
# Scan a package before installing
agentshield scan <package>==<version> --ecosystem pypi|npm|cargo

# Deep scan (downloads wheel, runs static analysis)
agentshield scan <package> --deep

# Scan all packages in a manifest file
agentshield scan-file requirements.txt
agentshield scan-file package.json
agentshield scan-file Cargo.toml

# Scan package + its full dependency tree
agentshield scan <package> --transitive

# Generate a CycloneDX SBOM from a manifest
agentshield sbom requirements.txt
agentshield sbom package.json -o sbom.json

# View async report log + installed package CVEs
agentshield posture

# Save HTML report
agentshield posture --format html --output report.html

# Run without network (fast)
agentshield scan <package> --offline

# Refresh vulnerability database
agentshield cache warm

# Check for drift in previously-allowed packages
agentshield drift-check

# Scan only packages that changed between two manifest snapshots
agentshield diff-scan old-requirements.txt new-requirements.txt

# Scan packages referenced in a Dockerfile
agentshield scan-docker Dockerfile

# Start MCP server (agents connect to this)
agentshield serve --mcp

# Start HTTP REST API server on localhost:8765
agentshield serve --http
agentshield serve --http --port 9000

# Start IPC daemon (shell scripts connect to this)
agentshield serve

# Start a guarded shell that intercepts pip/npm/cargo installs
agentshield guard
agentshield guard --shell zsh
```

## Troubleshooting

### `agentshield: command not found`

The pip bin directory is not on PATH.

```bash
# Find where pip installs scripts
python3 -m site --user-base
# Add <user-base>/bin to PATH in your shell profile
export PATH="$(python3 -m site --user-base)/bin:$PATH"
```

Or invoke via Python directly:
```bash
python3 -m agentshield scan requests
```

---

### `config.toml not found` / no config loaded

AgentShield uses defaults when `~/.config/agentshield/config.toml` does not exist — this is intentional and not an error. To confirm your config is being loaded:

```bash
agentshield cache stats
```

If you see an error about the DB path, the config file may have a syntax error. Validate with:
```bash
python3 -c "import tomllib, pathlib; tomllib.loads(pathlib.Path('~/.config/agentshield/config.toml').expanduser().read_text())"
```

---

### `cache warm` fails / times out

The OSV bulk download is large (~200–500 MB). If it times out:

```bash
# Warm only PyPI first (smallest download)
agentshield cache warm --ecosystems pypi
```

If OSV is unreachable, check your network. The OSV bulk export URL is `https://osv-vulnerabilities.storage.googleapis.com/<ECOSYSTEM>/all.zip`.

---

### MCP server not connecting

1. Confirm `agentshield serve --mcp` runs without error:
   ```bash
   echo '{"jsonrpc":"2.0","method":"ping","params":{},"id":1}' | agentshield serve --mcp
   ```
   Expected output line: `{"jsonrpc":"2.0","id":1,"result":{"pong":true}}`

2. Confirm the MCP client config file path is correct for your platform.

3. Restart the MCP client (Claude Desktop / Claude Code) after editing the config.

---

### Hermes / OpenClaw plugin not intercepting installs

1. Verify the module path is correct:
   ```bash
   python3 -c "from agentshield.integrations.hermes import AgentShieldPlugin; print('OK')"
   ```
   or
   ```bash
   python3 -c "from agentshield.integrations.openclaw import AgentShieldSkill; print('OK')"
   ```

2. Check that `agentshield[hermes]` / `agentshield[openclaw]` was installed (not just `agentshield`).

3. Ensure the Hermes/OpenClaw agent was **restarted** after editing the config file.

4. **If Hermes runs install commands via a `bash` or `shell` tool** and the plugin still doesn't fire, confirm that the tool name used by Hermes appears in `AgentShieldPlugin.intercepts`:
   ```python
   from agentshield.integrations.hermes import AgentShieldPlugin
   print(AgentShieldPlugin.intercepts)
   # ['bash', 'cargo_add', 'execute', 'npm_install', 'pip_install', 'run_command', 'shell', 'terminal']
   ```
   If your Hermes agent uses a different tool name (e.g. `run_bash`), subclass the plugin and add the name to `intercepts` as described in the setup section above.

---

### NVD / GitHub Advisory not returning results

These require API keys. Without keys:
- NVD rate-limits to 5 requests per 30 seconds and may time out under load
- GitHub Advisory returns no results at all

Set keys as described in Step 3 and retry. AgentShield logs a WARNING-level message when a source times out (it continues with remaining sources rather than failing).

To see these warnings:
```bash
PYTHONPATH=src python3 -c "
import logging, asyncio
logging.basicConfig(level=logging.WARNING)
from agentshield import AgentShield, ScanRequest, Ecosystem
r = AgentShield().scan(ScanRequest(package='requests', ecosystem=Ecosystem.PYPI))
print(r.decision.action)
"
```

---

### Posture report shows 0 packages scanned

The posture scanner enumerates packages visible to the Python interpreter running `agentshield`. If your agent uses a different virtualenv, run agentshield inside that virtualenv:

```bash
# Activate the agent's virtualenv first
source /path/to/agent/.venv/bin/activate
agentshield posture
```

Alternatively, use `--skip-packages` to skip the package scan and only show the async report log:
```bash
agentshield posture --skip-packages
```

## What's next

Once AgentShield is running:

1. **Review the posture report weekly:** `agentshield posture --format html -o posture.html`
2. **Keep the CVE database fresh:** `agentshield cache warm` (cron job or weekly manual run)
3. **Tune the config:** Add frequently-used trusted packages to `[allowlist]` to reduce noise
4. **Enable stricter policies** for production environments by setting `high = "block"` in `[defaults]`
